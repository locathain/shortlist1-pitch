Appendix17 · The unified lake
17 / 21

One lake. Three zones. One contract.

Microsoft Fabric on OneLake is the strategic landing zone. Bronze · Silver · Gold is the Microsoft reference pattern — and Fabric's Materialized Lake Views make the refresh story automatic in 2026.

Bronze

Raw landings

Source-shaped mirrors. SAP deltas, Salesforce objects, Odoo extracts, CStack tables, document binaries. Delta Parquet, partitioned by source. No transformations.

Silver

AI-resolved · canonical

Cleansed, deduplicated, mastered. Profisee 2026.R1 AI Similarity Matching resolves duplicates fuzzy strings cannot. Survivorship picks golden records. Stewards approve edge cases.

Gold

Business-ready 360s

Customer 360, Property 360, Unit 360, Lease 360, Contract 360. Aligned to OSCRE Industry Data Model. Consumable by Power BI Direct Lake, AI Search indexers, the graph builder.

Why Fabric, not Synapse

Microsoft has consolidated its data strategy on Fabric for new workloads. Synapse Dedicated Pools are de-emphasised. Direct Lake gives Power BI sub-second performance on Gold without copy. Databricks-on-Azure can read OneLake natively via Unity Catalog.

Appendix18 · AI mapping — entity resolution
18 / 21

Records that mean the same thing. Not records that look the same.

In February 2026 Profisee shipped AI Similarity Matching using vector embeddings. Fuzzy strings could not solve your 33,500 broken records. This can.

The problem · before

Why 33,500 records resist deterministic matching

  • Transliteration drift — "محمد", "Mohammed", "Mohamed", "Mohd" map to the same person; exact-string fails
  • Address normalisation — "Bldg 7", "Tower 7", "برج ٧", "Block 7 Unit A" — all the same unit
  • Merged-entity history — subsidiaries acquired with their own customer bases collide on national ID
  • Field-level semantic drift — SAP "Customer", Salesforce "Account", Odoo "Partner"
The solution · 2026

Three Microsoft-supported techniques, layered

  • Vector-embedding entity resolution — Profisee 2026.R1 generates embeddings per record; cosine similarity catches semantic duplicates
  • Azure OpenAI semantic field mapping — automated mapping of source attributes to canonical OSCRE schema
  • Profisee Aisey AI agents — autonomous data-quality and enrichment with full audit trail
  • Steward workflow — ambiguous cases route to a human, never auto-merged
Appendix19 · Hybrid retrieval — search + graph
19 / 21

Two surfaces. One substrate. Three-times accuracy.

Vector search excels at similar text. A knowledge graph excels at traversing relationships. Microsoft GraphRAG combines them — benchmarking 3× accuracy gains on connected-evidence questions.

Surface A

Azure AI Search

Hybrid retrieval — exact keyword match, semantic (meaning-based) match, and a final re-ranker, in one query.

Integrated vectorization — chunking and embedding inside the indexer. No external ETL.

Skillsets over LaserFiche, Archdesk and Azure OCR — extract entities, key phrases, relationships from your document estate.

Native permission-trimming via Entra ID — every result respects the caller's permissions. New in 2025.

Surface B

Knowledge graph + GraphRAG

Relationships as first-class citizens — Customer ↔ Unit ↔ Property ↔ Contract ↔ Subsidiary, with provenance per edge.

Microsoft GraphRAG — community detection and hierarchical summarisation; answers "global" questions vector search cannot.

Neo4j AuraDB on Azure — Microsoft Agent Framework has a first-class Neo4j GraphRAG context provider.

Alternative — Cosmos DB for NoSQL using Microsoft's AI Knowledge Graph pattern.

Local context from vector · global synthesis from graph · up to 3× accuracy on cross-evidence questions (Microsoft Research · 54.2% vs 16.7% baseline RAG)
